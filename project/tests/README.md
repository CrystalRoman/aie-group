# Папка `tests`

В этой папке лежат простые sanity-check и unit-тесты проекта.

## Какие тесты здесь находятся

### `test_metrics.py`
Проверяет базовую корректность вычисления метрик:
- `Dice`
- `IoU`

Тесты построены на простых искусственных масках и нужны, чтобы убедиться, что метрики не сломаны.

### `test_model_forward.py`
Проверяет, что модель создаётся и делает `forward` с корректной формой выхода.

### `test_dataset.py`
Проверяет, что `PneumothoraxDataset`:
- читает sample-данные;
- возвращает тензоры;
- не падает на базовом вызове.

### `test_config_loading.py`
Проверяет корректную загрузку YAML-конфига.

### `conftest.py`
Добавляет корень проекта в `sys.path`, чтобы `pytest` видел модуль `src`.

## Как запускать тесты

Из корня проекта:

```bash
pytest tests -v
```

На Windows:

```bat
python -m pytest tests -v
```

В Datasphere:

```python
import sys, subprocess
subprocess.run([sys.executable, "-m", "pytest", "tests", "-v"], check=False)
```

## Роль этой папки

Тесты здесь не предназначены для полного end-to-end обучения на большом датасете.  
Их задача — быстро проверить, что проект не сломан на базовом уровне:
- данные читаются;
- модель создаётся;
- метрики считаются;
- конфиг загружается.
