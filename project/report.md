# Отчёт по итоговому проекту по курсу «Инженерия Искусственного Интеллекта»

## 1. Паспорт проекта

- **Название проекта:** Сравнительный анализ архитектур нейронных сетей для сегментации пневмоторакса
- **Автор:** Савельев Роман Сергеевич
- **Группа:** РТСИИ-01-24
- **Контакт:** `<дописать>`
- **Ссылка на репозиторий:** `<дописать>`

Проект посвящён задаче автоматической сегментации пневмоторакса на рентгеновских снимках грудной клетки.  
В рамках работы были реализованы, обучены и сравнены три архитектуры: `UNet`, `AttentionUNet` и `TransformerUNet`.  
Итоговая цель — получить воспроизводимый ML-пайплайн с обучением, артефактами, сравнением моделей по метрикам и минимальным API для инференса.

## 2. Постановка задачи и контекст

Предметная область проекта — медицинская визуализация.  
Требуется по рентгеновскому снимку грудной клетки построить бинарную маску области пневмоторакса.

### Формулировка задачи в терминах ML

- **Вход:** рентгеновское изображение грудной клетки
- **Выход:** бинарная маска пневмоторакса
- **Тип задачи:** семантическая сегментация
- **Ограничения:** устойчивое обучение на сложных и малых масках, воспроизводимый пайплайн, корректный инференс

### Целевые метрики

- `Dice`
- `IoU`

Эти метрики выбраны как основные, потому что они отражают качество совпадения предсказанной и истинной масок и являются стандартными для медицинской сегментации.

Основные шаги:
- чтение CSV;
- сопоставление изображений и масок;
- фильтрация отсутствующих и некорректных масок;
- вычисление `mask_coverage`;
- ресайз и аугментации;
- positive-only режим и фильтрация маленьких масок в финальной схеме.

Ноутбук EDA:

Важные наблюдения:
- при смешанном обучении модель была склонна к пустым предсказаниям;
- финальная устойчивая схема получилась при positive-only режиме и фильтрации маленьких масок.

## 5. Экспериментальный протокол и результаты

### Экспериментальный протокол

- `stage_1_train_images.csv` использовался для `train/val`;
- `stage_1_test_images.csv` использовался как внешний test;
- сравнение моделей проводилось при одинаковой схеме обучения;
- лучшие веса сохранялись в `artifacts/.../models/`;
- история обучения сохранялась в `history_*.csv`.

### Сравнение моделей

| Модель | Конфиг | Лучший `val_dice` | Лучший `val_iou` | Комментарий |
|---|---|---:|---:|---|
| UNet | `configs/training_unet.yaml` | 0.4823 | 0.3454 | Baseline |
| AttentionUNet | `configs/training_attention_unet.yaml` | 0.5093 | 0.3738 | Улучшение относительно baseline |
| TransformerUNet | `configs/training_transformer_unet.yaml` | 0.5737 | 0.4298 | Лучшая итоговая модель |

### Выбор финальной модели

Финальной моделью выбран `TransformerUNet`, так как он показал лучшие значения `Dice` и `IoU` по сравнению с `UNet` и `AttentionUNet`.

## 6. Архитектура решения и сервис

### Архитектура пайплайна

1. чтение и фильтрация данных;
2. построение dataset/dataloader;
3. обучение модели;
4. сохранение чекпоинтов и метрик;
5. инференс через FastAPI.

### API и endpoints

Реализованы endpoints:
- `GET /health`
- `POST /predict`
- `POST /reload`

Что делает `/predict`:
- принимает PNG/JPG файл;
- прогоняет снимок через реальную обученную модель;
- строит бинарную маску;
- сохраняет mask и overlay;
- возвращает JSON с информацией о результате.

Параметры `/predict`:
- `file`
- `threshold` — необязательный параметр бинаризации

Возвращает:
- имя модели;
- threshold;
- число positive пикселей;
- долю маски;
- пути к `mask` и `overlay`;
- URL до сохранённых файлов.

Сервисные файлы:
- `src/service.py`
- `configs/service.yaml`
- `notebooks/service_api_check.ipynb`

### Как работает threshold в сервисе

Сервис по умолчанию использует лучший подобранный threshold для текущей модели, сохранённый в артефактах обучения.  
Этот threshold автоматически подставляется и в Swagger UI. Пользователь может изменить его вручную только в допустимом диапазоне.
