# Папка `data`

В этой папке хранится небольшой sample-поднабор данных, необходимый для демонстрации структуры проекта, EDA и базовых запусков.

## Что здесь может лежать

В текущем проекте в папке `data/` ожидаются:

- `stage_1_train_images.csv`
- `stage_1_test_images.csv`
- `png_images/`
- `png_masks/`
- `README.md`
- `data_description.md`

## Назначение файлов

### `stage_1_train_images.csv`
CSV с метаданными для обучающей выборки.

### `stage_1_test_images.csv`
CSV с метаданными для тестовой выборки.

### `png_images/`
Папка с рентгеновскими изображениями в формате PNG.

### `png_masks/`
Папка с бинарными масками пневмоторакса в формате PNG.

## Почему в репозитории только sample

Полный датасет может быть слишком большим для git-репозитория.  
Поэтому в итоговый проект лучше включать только:
- небольшой sample;
- структуру файлов;
- описание данных;
- инструкции по работе с полным набором при необходимости.

## Как использовать эту папку

- training-конфиги могут ссылаться на `data/...`, если проект должен запускаться полностью из репозитория;
- ноутбуки EDA и проверка сервиса могут использовать sample-данные из этой папки;
- FastAPI `/predict` можно тестировать на изображениях из `data/png_images/`.
