# Папка `artifacts`

В этой папке хранятся артефакты, полученные в результате обучения, оценки и демонстрации моделей.

## Что здесь может лежать

В зависимости от конфигурации и модели внутри `artifacts/` могут находиться:

- веса моделей (`best_*.pt`);
- таблицы и файлы с метриками (`history_*.csv`, `train_summary_*.json`);
- графики обучения и оценки (`loss_curve_*.png`, `dice_curve_*.png`, `threshold_search_*.png`);
- prediction examples (`*_overlay.png`, `*_mask.png`, итоговые иллюстрации качества);
- логи обучения (`train.log`);
- результаты сервисного инференса (`service_predictions/`).

## Как устроена папка в этом проекте

Для удобства сравнения моделей артефакты разложены не по типу файла, а по экспериментам / моделям.  
Это позволяет открыть одну папку и сразу увидеть всё, что относится к конкретной модели.

Пример структуры:

```text
artifacts/
├─ unet_positive_tversky/
│  ├─ models/
│  ├─ metrics/
│  ├─ figures/
│  └─ logs/
├─ attention_unet_positive_tversky/
│  ├─ models/
│  ├─ metrics/
│  ├─ figures/
│  └─ logs/
├─ transformer_unet_positive_tversky/
│  ├─ models/
│  ├─ metrics/
│  ├─ figures/
│  └─ logs/
└─ service_predictions/
```

## Что лежит внутри подпапок

### `models/`
Содержит финальные веса модели, например:
- `best_unet.pt`
- `best_attention_unet.pt`
- `best_transformer_unet.pt`

Обычно для демонстрации и сервиса нужен именно файл `best_*.pt`.

### `metrics/`
Содержит численные результаты обучения и оценки:
- `history_*.csv` — история обучения по эпохам;
- `train_summary_*.json` — итоговые метрики, размер выборок, подобранный threshold;
- `train_split.csv`, `val_split.csv`, `test_split.csv` — сохранённые разбиения.

### `figures/`
Содержит графики и изображения:
- кривые loss/dice/iou;
- результаты поиска threshold;
- prediction examples;
- изображения масок и overlay.

### `logs/`
Содержит текстовые логи обучения, чтобы можно было понять:
- какой конфиг запускался;
- где остановилось обучение;
- какой checkpoint был лучшим;
- не было ли ошибок в процессе.

### `service_predictions/`
Эта папка используется FastAPI-сервисом.  
При вызове `/predict` туда сохраняются:
- `*_mask.png`
- `*_overlay.png`

## Примечание о старых путях

Если в некоторых старых артефактах встречаются пути вида `data_full/...`, это исторические пути прошлых запусков на полном датасете.  
Такие файлы являются сохранёнными результатами экспериментов и не используются как активные конфиги проекта.

## Примечание о весах моделей

Для воспроизводимой демонстрации сервиса в репозитории используется вес `UNet`.
Веса `AttentionUNet` и `TransformerUNet` не включены в git-репозиторий из-за большого размера, но могут быть предоставлены отдельно.
