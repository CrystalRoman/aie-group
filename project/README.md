## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - `src/data/` — загрузка и подготовка датасета;
  - `src/models/` — архитектуры моделей;
  - `src/training/` — цикл обучения, loss-функции и метрики;
  - `src/train.py` — точка входа для запуска обучения.
- `data/` – описание данных и sample-структура.
- `configs/` – YAML-конфиги обучения.
- `tests/` – базовые тесты и sanity-check проверки.
- `artifacts/` – сохранённые модели, метрики, графики и логи.

Если структура проекта отличается от базовой, опишите это здесь.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10` (уточните при необходимости).
- CUDA-совместимое окружение при запуске обучения на GPU
- Доступ к библиотекам `torch`, `torchvision`, `pandas`, `numpy`, `matplotlib`, `scikit-learn`, `PyYAML`

## 4. Как запустить проект

Опишите, как именно воспроизвести основные сценарии использования проекта.

Примеры (заполните под свой случай):

### 4.1. Подготовить данные

Полный датасет не хранится в репозитории. Локально должна существовать структура вида:

```text
data_full/
├─ stage_1_train_images.csv

```

CSV-файлы содержат как минимум колонки:

- `ImageId`
- `has_pneumo`

### 4.2. Настроить конфиг

В папке `configs/` лежат отдельные конфиги для разных моделей.  
Перед запуском нужно проверить пути:

- `train_csv_path`
- `test_csv_path`
- `images_dir`
- `masks_dir`
- `artifacts_dir`

### 4.3. Запуск обучения

Пример запуска UNet:

```bash
python -m src.train --config configs/training_unet.yaml
```

Пример запуска AttentionUNet:

```bash
python -m src.train --config configs/training_attention_unet.yaml
```

Пример запуска TransformerUNet:

```bash
python -m src.train --config configs/training_transformer_unet.yaml
```

### 4.4. Запуск в Jupyter / Datasphere

```python
import sys, subprocess

cmd = [sys.executable, "-u", "-m", "src.train", "--config", "configs/training_unet.yaml"]
process = subprocess.Popen(cmd, stdout=subprocess.PIPE, stderr=subprocess.STDOUT, text=True, bufsize=1)

for line in process.stdout:
    print(line, end="")

print(f"\nProcess finished with code: {process.wait()}")
```

## 5. Данные

Для обучения использовался открытый датасет, подготовленный на основе соревнования **SIIM-ACR Pneumothorax Segmentation**. В проекте использовались PNG-изображения рентгеновских снимков грудной клетки и бинарные маски пневмоторакса. В предыдущей практике по этой теме использовался датасет объёмом 12 047 изображений и масок размером 1024×1024, а также отмечался сильный дисбаланс классов — около 70% снимков без патологии. В текущем проекте для удобства обучения применялись CSV-индексы `stage_1_train_images.csv` и `stage_1_test_images.csv`. 

В репозитории рекомендуется хранить:
- только sample-данные;
- описание структуры данных;
- инструкцию по скачиванию или подключению полного набора.

Большие реальные данные в репозиторий не добавляются.

## 8. Ограничения и дальнейшая работа

Ограничения текущей версии:
- проект ориентирован на исследовательскую часть, а не на API/сервис;
- использовалась упрощённая постановка с `positive_only` в финальных экспериментах;
- результаты чувствительны к фильтрации очень маленьких масок и выбору threshold;
- финальный выбор делался по validation-метрикам, а не по полноценной внешней клинической оценке.

Что можно улучшить дальше:
- добавить полноценный inference-скрипт или сервис `/predict`;
- провести отдельную оценку на внешнем test-наборе с более широкими метриками;
- реализовать более полный набор тестов;
- попробовать более сложные аугментации и loss-функции;
- добавить анализ uncertainty и post-processing масок.
