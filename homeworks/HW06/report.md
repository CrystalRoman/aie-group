## 1. Dataset

- Какой датасет выбран: `S06-hw-dataset-04.csv`
- Размер: (25000, 62)
- Целевая переменная: `target` (класс 0: 95.08%, класс 1: 4.92%)
- Признаки: Все признаки числовые (f01-f60)

## 2. Protocol

Разбиение: 80% train, 20% test, `random_state=42`, `stratify=y` для сохранения пропорций классов

Подбор: 3 фолда, StratifiedKFold, оптимизация по ROC-AUC (уместно для несбалансированных данных)

Метрики: accuracy, F1, ROC-AUC, Precision, Recall, Average Precision
- ROC-AUC - основная метрика, так как устойчива к дисбалансу классов
- F1 и Average Precision важны для оценки качества на миноритарном классе
- Accuracy дает общее представление, но может вводить в заблуждение при дисбалансе

## 3. Models

Сравнивались следующие модели:

1. DummyClassifier (baseline) - предсказывает наиболее частый класс
2. LogisticRegression (baseline) - линейная модель с нормализацией
3. DecisionTreeClassifier - с подбором max_depth и min_samples_leaf
4. RandomForestClassifier - с подбором n_estimators, emax_depth и min_samples_leaf
5. HistGradientBoostingClassifier - градиентный бустинг с гистограммами
6. StackingClassifier - композиция моделей с логистической регрессией как метамоделью

## 5. Analysis

Устойчивость:

- Проведено 5 прогонов с разными random_state
- ROC-AUC для HistGradientBoosting колеблется в пределах 0.895-0.912
- Стандартное отклонение метрик низкое (<0.01), что указывает на устойчивость модели

Сonfusion matrix:

- True Negative (TN): 4749 (модель правильно определила негативные случаи)
- False Positive (FP): 5 (модель ошибочно предсказала позитивные случаи)
- False Negative (FN): 103 (модель ошибочно предсказала негативные случаи)
- True Positive (TP): 143 (модель правильно определила позитивные случаи)

Ошибки:

- Confusion matrix показывает хороший баланс: высокая точность (97.79%) при умеренной полноте (57.65%). Модель делает всего 5 ложных срабатываний (FP), что критично для задач обнаружения аномалий, но пропускает 42% реальных позитивных случаев (FN). Такой компромисс приемлем для практических применений, где ложные срабатывания дороже пропущенных случаев.

Интерпретация:

- Топ-3 важных признаков: f58, f25, f38, f54, f53, f47, f13, f04, f33, f11, f08, f41, f57, f43, f16
- Важность признаков стабильна при различных запусках
- Некоторые признаки имеют явное влияние на классификацию, что соответствует синтетической природе данных

## 6. Conclusion

1. Деревья решений легко переобучаются на несбалансированных данных, что видно по падению ROC-AUC при увеличении сложности дерева.
2. Ансамбли (особенно бустинг) значительно превосходят одиночные деревья и линейные модели при работе с несбалансированными данными.
3. ROC-AUC и Average Precision - более информативные метрики, чем accuracy, для оценки качества моделей на несбалансированных данных.
4. Правильная настройка гиперпараметров через кросс-валидацию критически важна для получения надежных результатов.
5. Permutation importance позволяет интерпретировать сложные модели и выявить ключевые признаки.
6. Честный ML-эксперимент требует строгого разделения на train/test, фиксации random_state и документирования всех этапов.
