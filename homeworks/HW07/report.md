## 2. Protocol

Препроцессинг: 
- Для всех датасетов использовал StandardScaler для масштабирования числовых признаков
- Пропусков не было, поэтому SimpleImputer не использовался
- Категориальных признаков не было, поэтому кодирование не требовалось

Поиск гиперпараметров:
- Для KMeans: проверял k от 2 до 10, 15, 20
- Для DBSCAN: проверял eps в диапазоне [0.4, 1.2] и min_samples в [3, 5, 7, 10]
- Выбор "лучшего": основывался на silhouette score (чем выше, тем лучше)

Метрики: 
- silhouette_score (выше - лучше)
- davies_bouldin_score (ниже - лучше)
- calinski_harabasz_score (выше - лучше)
- Для DBSCAN: метрики считались на non-noise точках, доля шума фиксировалась отдельно

Визуализация: 
- PCA(2D) для визуализации кластеров
- Графики зависимости метрик от гиперпараметров

## 3. Models

Для каждого датасета:
- KMeans (поиск `k` от 2 до 10, 15, 20; фиксировали `random_state=42`, `n_init=50`)
- DBSCAN (`eps` от 0.4 до 1.2, `min_samples` [3,5,7,10], доля шума)

## 4. Results

Для каждого датасета – краткая сводка результатов.

### 4.1 Dataset A

- Лучший метод: KMeans (k=4)
- Метрики: silhouette=0.5216, Davies-Bouldin=0.6853, Calinski-Harabasz=11786.95
- KMeans идеально подходит для этого датасета после масштабирования признаков, так как данные образуют четкие сферические кластеры. DBSCAN показал худшие метрики, так как он менее эффективен для данных с равномерной плотностью и четкими границами между кластерами. Высокое значение Calinski-Harabasz (11786.95) подтверждает хорошее разделение кластеров.

### 4.2 Dataset B

- Лучший метод: DBSCAN (eps=0.5, min_samples=5)
- Метрики: silhouette=0.4138, Davies-Bouldin=0.5997, Calinski-Harabasz=48.12
- Доля шума 12.4%. Шумовые точки корректно идентифицированы и исключены из основных кластеров.
- DBSCAN значительно лучше справился с нелинейной структурой данных. Визуализация показывает, что KMeans искусственно разделил естественные группы точек из-за своей предрасположенности к сферическим кластерам. DBSCAN смог выявить естественные границы данных и корректно обработать выбросы, поместив их в категорию шума, что соответствует характеристикам этого датасета.

### 4.3 Dataset C

- Лучший метод: DBSCAN (eps=0.7, min_samples=5)
- Метрики: silhouette=0.3726, Davies-Bouldin=0.5511, Calinski-Harabasz=17.20
- Доля шума 18.5%. Значительная часть точек была корректно идентифицирована как фоновый шум.
- DBSCAN лучше справляется с кластерами разной плотности, что характерно для этого датасета. Визуализация подтверждает, что KMeans принудительно разделил естественно связанные группы точек на сферические кластеры, в то время как DBSCAN сохранил целостность структур разной плотности. Несмотря на низкое значение Calinski-Harabasz (17.20), низкий Davies-Bouldin (0.5511) и визуальная согласованность кластеров подтверждают преимущество DBSCAN для этого типа данных.

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

- KMeans "ломается" на данных с нелинейной структурой (Dataset B) и кластерами разной плотности (Dataset C), так как алгоритм предполагает сферические кластеры одинакового размера и плотности.
- DBSCAN выигрывает на Dataset B и C благодаря способности находить кластеры произвольной формы и обрабатывать шум.
- Масштабирование признаков было критически важным для всех датасетов, так как признаки были в разных шкалах.

### 5.2 Устойчивость (обязательно для одного датасета)


- Проверка устойчивости: выполнено 5 запусков KMeans с разными random_state для Dataset A (k=4) после масштабирования признаков.
- Результаты: средний ARI = 1.000. Это указывает на полную идентичность кластерных разбиений при всех запусках независимо от начальной инициализации центроидов.
- Вывод: решение абсолютно устойчиво. Такой результат объясняется четкой сферической структурой кластеров в Dataset A после правильного масштабирования признаков. Границы между кластерами настолько однозначны, что алгоритм всегда сходится к одному и тому же оптимальному решению, что подтверждает корректность выбора KMeans для этого датасета.
